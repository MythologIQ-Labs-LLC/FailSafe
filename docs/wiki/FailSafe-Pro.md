# FailSafe Pro vs FailSafe

FailSafe is available in two editions: the open-source VS Code/Cursor extension and the desktop-native Pro application.

## Comparison

| Feature | FailSafe (Open Source) | FailSafe Pro |
|---------|----------------------|--------------|
| **Platform** | VS Code / Cursor extension | Desktop native application |
| **Governance** | Editor-level governance | Full SDLC visibility |
| **Enforcement** | Save-time governance | OS-level enforcement + file locking |
| **Scope** | Editor boundary | Beyond the editor boundary |
| **Teams** | Single-user | Team workflows |
| **Connections** | Local-first | Remote connections |
| **UI** | VS Code webview + sidebar | Native desktop UI |
| **Price** | Free (Apache 2.0) | Commercial |

## When to Use Which

### Use FailSafe When:

- You want local editor guardrails for AI coding
- You use VS Code, Cursor, or compatible editors
- You need governance for individual development
- You want open-source, extensible governance
- You're evaluating AI safety for the first time

### Use FailSafe Pro When:

- You need full SDLC visibility and governance
- You require OS-level enforcement and file locking
- You're managing team workflows
- You need remote connections beyond the editor
- You require managed runtime operations
- You need enterprise compliance features

## FailSafe Pro Features

- **OS-level enforcement**: Governance at the operating system level, not just the editor
- **File locking**: Prevent unauthorized modifications to critical files
- **Team workflows**: Shared governance policies across team members
- **Remote connections**: Governance for remote development environments
- **Desktop native**: Full native UI experience with system integration

## Integration

FailSafe and FailSafe Pro are designed to complement each other. Teams can use:

- **FailSafe** for individual developer governance in the editor
- **FailSafe Pro** for organizational governance and compliance

Both share the same QoreLogic governance engine and SHIELD lifecycle.
